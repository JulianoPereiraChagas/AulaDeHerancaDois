Programção orientada à objeto = paradigma de programação que organiza o software em torno de "objetos"

Objeto = Molde da vida real transformado em código, uma instancia de uma classe

Classe = Define os atributos e métodos comuns que serão compartilhados por um objeto =

Instanciar = Criar objeto a partir de uma classe

Se tiver parenteses () = Método

Se não tiver parenteses () = Atributo

Eu digo que os objetos estão em estados diferentes, mas todos eles vieram da mesma classe

Abstração (considerado o primeiro pilar por muitos) = que foca em extrair e modelar apenas as características e comportamentos essenciais de um objeto para o seu sistema, ignorando detalhes irrelevantes

Modelo = String = Bloco de texto/Armazenar textos, armazenamento de palavras ou frases inteiras

Boolean = tomada de decisões

Visibilidade = determina onde variáveis e métodos podem ser acessados ou alterados no código

Diagrama de classes = modelo visual da estrutura estática de um sistema

+ = Público = A classe atua e todas a outras classes

- = Privado = A classe atual

# = Protegido = A classe atual e todas as outras sub classes

Métodos acessores/getter = métodos que dão acesso à uma determinada coisa

Métodos modificadores/setter = modificam coisas que estão dentro do objeto

Num = Number/Número
